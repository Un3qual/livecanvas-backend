# Profiles And Social Basics Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the profile placeholder with Relay-backed viewer and other-user profile surfaces, privacy controls, and the first relationship inbox actions supported by the current mobile GraphQL contract.

**Architecture:** Keep profile data in Relay queries and mutations inside the signed-in `(app)` route group. Use small pure presentation helpers for sparse backend data, keep unauthorized/private states explicit, and avoid inventing profile fields that the backend does not expose yet. Treat missing backend contract for unfollow or richer profile customization as a reported dependency, not a mobile workaround.

**Tech Stack:** Expo Router, React Native, Relay, generated GraphQL artifacts, existing auth/viewer bootstrap providers, Bun tests for pure helpers, TypeScript verification.

---

## Current State Verification

Verified before writing this plan:

1. The Relay/auth/session lifecycle plan is complete through Task 5 and Task 6 verification.
2. `mobile/app/(app)/profile.tsx` is still a placeholder card.
3. `ViewerBootstrap` already fetches `viewer { id email privacyMode insertedAt }` and exposes `useViewer()`.
4. The mobile schema exposes viewer-scoped social reads: `viewerPendingFollowRequests`, `followers`, `following`, `relationshipState`, and `isMuted`.
5. The schema exposes `updateViewerPrivacyMode`, `followUser`, `acceptFollowRequest`, `declineFollowRequest`, `muteUser`, and `unmuteUser`.
6. The schema does not expose an `unfollow` mutation or rich profile fields such as username, display name, bio, or avatar.

## Scope Decisions

- **In scope:** viewer profile summary, privacy mode display/update, follower/following previews, inbound follow-request inbox, accept/decline request actions, other-user profile route, relationship state display, follow/request action where supported.
- **Out of scope:** unfollow, profile editing beyond privacy mode, avatar/name/bio editing, contact discovery, live-session playback, chat, Phoenix Channels, backend schema changes.
- **Backend dependency to report if encountered:** a product requirement for unfollow or richer profile fields needs a backend contract update before mobile implementation.
