# Mobile Expo Bootstrap Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Bootstrap a standalone Expo app in `mobile/` with the default
template, `pnpm`, and a committed Nix development shell.

**Architecture:** Keep the mobile frontend isolated inside `mobile/` so the
existing Elixir backend workspace stays untouched by repo-root JavaScript
tooling. Use an ad hoc Nix shell to run `pnpm dlx create-expo-app`, then add a
local `mobile/flake.nix` so future mobile commands run through a repeatable
Nix-managed environment.

**Tech Stack:** Nix, Expo, React Native, TypeScript, pnpm

---

## Scope Decisions

- Treat generated Expo boilerplate and Nix config as a TDD exception for this
  slice; verify with generator and tool commands instead of inventing fake
  regression tests around scaffolding.
- Keep the scaffold limited to the default Expo template plus a Nix shell and
  local usage instructions.
- Do not add repo-root frontend files or convert the backend repo into a
  pnpm workspace.
- After this slice, hand `docs/plans/NOW.md` back to the interrupted backend
  batch unless the user extends mobile execution.
